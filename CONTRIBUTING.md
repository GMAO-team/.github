# Contributing

## Development flow
1. Fork → feature branch (`feat/...`, `fix/...`).
2. Run `pre-commit run -a` and tests.
3. Small PRs, with a clear description and screenshots where relevant.

## Standards
- Code style: **Black**
- Lint: **Ruff**
- Commits: Conventional (`feat:`, `fix:`, `docs:`, `chore:`…)
- Reviews: 1 maintainer approval required.
